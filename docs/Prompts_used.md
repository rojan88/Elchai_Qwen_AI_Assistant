# Prompts Used

The following prompts are used while designing and testing the proof-of-concept project.

--
## Prompt 1

Read the uploaded business documents and generate a concise summary highlighting the most important information.

---

## Prompt 2

Extract key information from the uploaded document, including department,document type, important dates,action items, and overall purpose.

---

## Prompt 3

Identify which department should handle this document:
- Engineering
- Finance
- Product
- Operations
- IT
- strategy

Provide a short explanation.

--

## Prompt 4

Generate a list of action items from the uploaded document.

---

## Prompt 5

Rewrite the document summary in a professional business format suitable for management review.

---

## Development Tools

- Qwen (selected open-source model for research)
- python
- Streamlit
- HTML
- CSS
- Javascript
- ChatGPT (used for research assistance, documentation drafting, and project planning)

---

## Intended Alternative

If direct access to Qwen is unavailable, an equivalent open-sourced model such as kimi, GLM, or MiniMax can be evaluated using the same workflow and prompts.
